What It Does

mlx-manager is a CLI tool that manages local MLX language-model HTTP servers on Apple Silicon Macs. It lets you start, stop, and monitor inference servers from the command line — designed specifically for headless SSH-only machines where GUI tools don't work.

Server Lifecycle

Start, stop, and restart MLX servers with full PID tracking, readiness probes, and automatic log rotation. Safety checks prevent killing unrelated processes.

Model Discovery

Automatically finds models across three layouts: flat directories, LM Studio nested trees, and HuggingFace cache. Supports custom aliases for friendly model names.

Provider Integration

Generates ready-to-use configuration snippets for OpenCode (primary) and Claude Code / LiteLLM (secondary). One command to integrate your local model with AI coding tools.

Performance Benchmarking

Measures TTFT (time-to-first-token), decode throughput, and aggregate parallel-stream performance. Supports concurrent requests for realistic load testing.

Safety Mechanisms

PID Reuse Protection

  • Verifies PID is alive before any action
  • Checks argv contains mlx_lm.server
  • Confirms recorded port appears in argv
  • Refuses to kill unrelated processes

Concurrency Control

  • fcntl exclusive file locks on state mutations
  • 10-second acquisition timeout
  • Automatic lock release on completion or error

Atomic Operations

  • State file writes use write-tmp + rename pattern
  • Config apply writes .bak backup before modifying
  • Log rotation preserves history within limits

Quick Start

pip install -e .
mlx-manager doctor              # check Python, mlx_lm, paths, port
mlx-manager list                # see discovered models
mlx-manager start --model qwen3-8b-4bit   # launch server
mlx-manager status              # see live PID, uptime, endpoint
mlx-manager config opencode --model qwen3-8b-4bit --apply
mlx-manager benchmark --concurrency 4       # load test
mlx-manager stop

Configuration

Config file at ~/.config/mlx-manager/config.toml (auto-created on first run). All paths expand ~ and $VAR.

[server]
host = "127.0.0.1"              # Default bind address
port = 8080                      # Default port
log_file = "~/services/mlx/logs/mlx-lm.server.log"
startup_timeout_seconds = 120    # Max wait for readiness
max_log_bytes = 10485760         # 10 MiB rotation threshold

[models]
directories = [                  # Roots to scan for models
  "~/models/mlx",
  "~/.cache/huggingface/hub",
  "~/.lmstudio/models",
]

[providers]
api_key = "mlx-local"            # Dummy key for local provider
provider_name = "mlx-local"

Provider Integration

OpenCode (Primary)

Generates JSON provider blocks for opencode.json. Supports merge mode (preserves user-tuned fields) and overwrite mode.

{
  "provider": {
    "mlx-local": {
      "npm": "@ai-sdk/openai-compatible",
      "options": {
        "baseURL": "http://127.0.0.1:8080/v1"
      }
    }
  }
}

Claude Code / LiteLLM

Claude Code doesn't support OpenAI-compatible routing yet. Two paths offered:

Recommended: LiteLLM proxy in front of MLX

model_list:
  - model_name: mlx-local/qwen3-8b-4bit
    litellm_params:
      model: openai/qwen3-8b-4bit
      api_base: http://127.0.0.1:8080/v1

Experimental: Direct env vars

export OPENAI_API_KEY="mlx-local"
export OPENAI_BASE_URL="http://127.0.0.1:8080/v1"
